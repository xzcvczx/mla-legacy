1.  What is Microchip Application Libraries?

Microchip Application Libraries is a collection of Microchip firmware libraries and demo projects. Not all firmware libraries and demo projects from Microchip are distributed in this package; rather, this package includes a few specific libraries that tend to be used together.

 

By distributing libraries that are used often together, Microchip can provide example projects that integrate the use of multiple libraries.

 

Some example projects that use multiple libraries are:

 

Graphics MultiApp Demo

PIC18F Starter Kit 1

PIC24F Starter Kit 1

PIC24H Starter Kit 1

TCPIP MDD Demo App

USB Host – Printer – Print Screen Demo

USB Host – Printer – Simple Full Sheet Demo

USB Host – Printer – Simple POS Demo

 

The Microchip Applications Libraries is currently being distributed mainly in the source code form to provide the maximum flexibility for end designers to customize firmware to fit his/her applications.

 

2.  Installing/Uninstalling

To obtain the latest version of the Microchip Application Libraries, you can download it from the Microchip web site at www.microchip.com/MLA.

 

Once the installation package is downloaded, first unzip (if zipped) the downloaded zip file and run the installer.

If installing over an existing Microchip Application Libraries folder location, files that will be overwritten are back-up to the folder named ‘rollbackBackupDirectory’.

One or more third party libraries maybe required to run PC programs, this includes but not be limited to Java Runtime Environment (JRE), Qt framework, and/or .NET framework.

 

For Java programs (.jar extension), the jar files must be associated with the JRE in order to run.

 

No MPLAB 8 projects are supported in the Mac and Linux distributions, even though you may see MPLAB 8 projects in the install. OS specific executables are installed in all distributions.

 

Some users may need to install multiple instances of the distribution; this can be done simply by selecting a different installation path and shortcut menu folder.

 

When upgrading from a previous version, it is recommended that users install the newer installation to a different installation path, and then consult each individual library release notes for changes that may impact application project. This method is safer than just overwriting the previous installation.

 

Microchip Application Libraries can be uninstalled by either running the uninstaller or simply delete the root installed folder. Either ways, users should make sure to back-up any modified files.

3.  What’s new/changed

 

This release includes support for MPLAB® XC16 and XC32 compilers. The minimum MPLAB® IDE versions required are 8.85 and X v1.20. There is a known issue with XC16 v1.00 that requires the workaround of defining -fno-ivopts option in the project compiler build setting. This disables the loop invariant optimizer. Other versions of XC16 will/do not require this workaround.

 

MPLAB X projects utilize the Build Configuration feature extensively. For each demo, it can run on many different hardware setups, select the appropriate build configuration setting for the supported hardware.

 

An important notice for the next Microchip Application Libraries release is, MPLAB 8 projects will no longer be distributed. Users can still use MPLAB 8 with the source code provided, but there will be no readymade 8 projects. Only MPLAB X projects will be available. Again, this is a planned migration in the next release.

3.1 MCHPFSUSB Framework

  • Device
    1. Added WHQL Certified CDC and WinUSB driver packages
    2. Added an appendix( page 876) section "Driver Signing and Windows 8( page 904)" to the MCHPFSUSB Library Help document. It is recommended that anyone designing or supporting a USB device that needs to work on Windows 8 read this section.
    3. Updated Low Pin Count USB Development Board( page 193) help documentation to provide usage tips when using the PIC16F145x microcontrollers on this platform
    4. Minor changes to improve PIC16F support and reduce RAM usage in WinUSB demos
  • Host
    1. Android files changed to request for protocol version and to wait for a user configurable startup delay.

 

For more information about the specific changes in this revision, please refer to the Revision History section of the MCHPFSUSB Library Help file available in the library download.

 

3.2 Graphics Library

 

  • Update Graphics Resource Converter (grc.jar).

 

For more information about the specific changes in this revision, please refer to the detailed release notes in the Graphics Library Help file available in the library download.

 

3.3MDD Library

  • Cleared the "read" flag and set the file pointer to NULL in FSfclose() function to prevent the unintentional acess to a closed file.
  • Modified "FSfopen()" function so that in "ReadPlus(r+)" mode the FAT table is read from media & the latest FAT contents are present in cache of RAM.
  • Modified "FILEget_next_cluster" function so that: if the last two clusters of the data region are allocated to a file, then that file can be traversed using "FILEget_next_cluster" function.
  • Added a check to the FSrename function to ensure that directory entries are renamed correctly.

 

For more information about the specific changes in this revision, please refer to the detailed release notes in the MDD Library Help file available in the library download.

3.4 TCP/IP Stack

  • Added new project WiFi G Demo (SoftAP mode support for now).
  • Added support for new security mode WF_SECURITY_WPA2_ENTERPRISE (EAP-PEAP/MSCHAPv2 and EAP-TTLS/MSCHAPv2).
  • Added mobile application demo and cloud networking demo.
  • Added new feature to support multi-client DHCP server.
  • Added new features to SoftAP network type to support active scan, created new event WF_EVENT_SOFT_AP_EVENT_SUBTYPE / WF_EVENT_SOFT_AP_EVENT to indicate client's status (connected or disconnected to softAP), supports Zeroconf & mDNS. Resolves wrong Zeroconf port issue. Search for keyword SOFTAP_ZEROCONF_SUPPORT.
  • The full versions of cryptographic files used by SNMPv3 and SSL (ARCFOUR.c, RSA.c, AES_PIC32MX.a) are now included in the primary stack distribution.  Users are no longer required to separately purchase the cryptographic libraries.  Distribution and redistribution of these files is subject to U.S. Export Restrictions; see the MLA landing page or the TCP/IP Stack Release notes for more information.

  

For more information about the specific changes in this revision, please refer to the detailed release notes in the TCPIP Stack Help file available in the library download.

3.5 mTouchCap Library

PIC12F PIC16F Library Bug Fixes:

  • Additional mComm register labels have been implemented that were previously missed. The labels can still be editted from the mComm_config.h file. There are now no known issues with using the 2nd or 3rd UART or I2C module.
  • The Hardware CVD acquisition file now correctly reloads the AADCAP value for each sensor regardless of the whether it is called out of the main loop or the ISR.
  • The Hardware CVD acquisition file now correctly calibrates the AADCAP value for the amount of capacitance seen on the pin.

 

PIC12F PIC16F Library Updates:

  • Reordered aquisition scanning to resolve issues with multiple scanning modes and unique oversampling values. The scan for one sensor is now always performed in one single interrupt. Previously the framework would interleave the scans of one sensor with the math of the previous - now the math and the scans are performed at the same time.
  • Removed support for PIC16F72x family due to reduced sensitivity of the 8-bit ADC.
  • Several unnecessary interrupt variables have been removed to save space.
  • PIC12LF1552 32MHz Fosc support added.
  • PIC16F1508/9 UART support added.

 

PIC18F PIC24F PIC24H dsPIC Library Bug Fixes:

  • The LogUINT16(…) function in mTouchDebug.c was fixed to correct the data for mTouch GUI.
  • The MTouchChargeDelay(chargeDelay) macro in mTouchAcquisitionMCU16.h was updated to resolve compiler and optimization dependency issue.

3.6 Smart Card Library

  • In SClib.c:

Changed the data type of variable "cgtETU" from "BYTE" to "unsigned short int".

Modified "SC_DoPPS" function, so as to add the guard time between transmission of bytes to smart card.

Modified "SC_CalculateWaitTime" function, so as to calculate correct guard & wait time values.

3.7 MiWi™ DE

  • None
  • More detailed release notes can be found under Microchip/WirelessProtocols/Documentation/Readme.txt

3.8Accessory Framework for Android™

  • Made changes to allow some tablets that do not follow the AOA protocol correctly to attach. Some devices appear to use the wrong VID/PID in AOA mode. Previously these devices would not work because the AOA driver was validating the VID/PID specified in the AOA protocol document.
